# 部署说明

本文档说明如何在本地或 openEuler / ARM 服务器上部署缺陷管理系统。

## 一、本地 Docker 部署

进入项目根目录：

```bash
cd bug-management-system
```

启动全部服务：

```bash
docker compose up -d --build
```

查看容器：

```bash
docker compose ps
```

查看日志：

```bash
docker compose logs -f
```

停止项目：

```bash
docker compose down
```

访问系统：

```text
http://localhost:8080
```

## 二、容器说明

| 容器 | 镜像 / 构建 | 说明 | 端口 |
|---|---|---|---|
| mysql | mysql:8.0 | 数据库 | 3306 |
| backend | backend/Dockerfile | Spring Boot 后端 | 8081 |
| frontend | frontend/Dockerfile | Nginx + Vue 前端 | 8080 |

前端 Nginx 会把接口请求转发给后端：

```text
/api -> backend:8081
```

## 三、openEuler / 鲲鹏 ARM 服务器部署

### 1. 检查系统信息

```bash
cat /etc/os-release
uname -m
```

如果显示：

```text
aarch64
```

说明服务器是 ARM 架构。

### 2. 安装基础工具

```bash
sudo dnf update -y
sudo dnf install -y git vim wget curl unzip tar net-tools lsof firewalld
```

### 3. 安装 Docker

```bash
sudo dnf install -y docker
sudo systemctl enable --now docker
sudo docker version
```

### 4. 检查 Docker Compose

```bash
sudo docker compose version
```

如果不可用，可以尝试安装旧版命令：

```bash
sudo dnf install -y docker-compose
docker-compose version
```

### 5. 开放端口

```bash
sudo systemctl enable --now firewalld
sudo firewall-cmd --permanent --add-port=8080/tcp
sudo firewall-cmd --reload
```

如需远程调试后端或数据库，也可以临时开放：

```bash
sudo firewall-cmd --permanent --add-port=8081/tcp
sudo firewall-cmd --permanent --add-port=3306/tcp
sudo firewall-cmd --reload
```

正式演示通常只需要开放 `8080`。

### 6. 上传或拉取项目

方式一：使用 git：

```bash
git clone <你的项目仓库地址>
cd bug-management-system
```

方式二：上传压缩包后解压：

```bash
unzip bug-management-system.zip
cd bug-management-system
```

### 7. 启动项目

```bash
sudo docker compose up -d --build
```

### 8. 查看容器状态

```bash
sudo docker compose ps
```

正常情况下应看到：

```text
bug-mysql
bug-backend
bug-frontend
```

### 9. 查看日志

```bash
sudo docker compose logs -f
```

单独查看后端日志：

```bash
sudo docker compose logs -f backend
```

### 10. 浏览器访问

本地访问：

```text
http://localhost:8080
```

服务器访问：

```text
http://服务器IP:8080
```

## 五、常见问题

### 1. Docker 命令不可用

检查 Docker 是否启动：

```bash
sudo systemctl status docker
sudo systemctl enable --now docker
```

### 2. 端口无法访问

检查防火墙：

```bash
sudo firewall-cmd --list-ports
```

确认已经开放 `8080/tcp`。

### 3. 后端连接不上 MySQL

查看容器状态：

```bash
sudo docker compose ps
```

查看 MySQL 日志：

```bash
sudo docker compose logs mysql
```

确认 MySQL 已经 healthy 后，再查看后端日志。

### 4. 修改初始化 SQL 后没有生效

MySQL 容器第一次启动后会把数据保存到 volume。若需要重新初始化数据库，可以删除 volume：

```bash
sudo docker compose down -v
sudo docker compose up -d --build
```

注意：`down -v` 会删除数据库数据。
